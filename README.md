# 🎮 Video Game Management - Java OOP Practice

## 📌 Overview

This project is a simple Object-Oriented Programming (OOP) practice exercise in Java.  
It models video games using a custom `VideoJuego` class and demonstrates core OOP concepts and collection handling.

---

## 🧱 Project Structure

### `VideoJuego` Class

Represents a video game with the following attributes:

- `codigo` – Unique identifier
- `titulo` – Game title
- `consola` – Console platform
- `cantJugadores` – Number of players
- `categoria` – Game category

Includes:

- Default and parameterized constructors
- Getters and setters
- Overridden `toString()` method

---

### `EjercicioVideojuego` (Main Class)

Demonstrates:

- Creating multiple `VideoJuego` objects
- Storing objects in an `ArrayList`
- Iterating through the list using enhanced for-loops
- Modifying object attributes
- Filtering games by console (Nintendo 64)

---

## 🚀 Features Demonstrated

- Object instantiation
- Encapsulation
- Constructor overloading
- Collection management using `ArrayList`
- Conditional filtering
- Method overriding
- Object property updates

---

## 🛠 Technologies Used

- Java (JDK 25)
- NetBeans IDE
- Java Collections Framework

---

## 🎯 Purpose

This project was created as part of Java OOP learning practice to strengthen understanding of:

- Class design
- Encapsulation
- Object interaction
- Basic collection handling
- Clean code structure

---

## ▶ How to Run

1. Open the project in NetBeans (or any Java IDE).
2. Run the `EjercicioVideojuego` main class.
3. View the console output to see object creation, updates, and filtering in action.

---

## 📚 Learning Focus

This exercise reinforces foundational Java concepts essential for building larger object-oriented systems.
